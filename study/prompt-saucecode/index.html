## 1. 기본 분석 원칙

분석할 때는 다음 순서를 지켜줘.

1. 먼저 전체 파일 구조를 파악한다.
2. 프로젝트의 목적과 기능을 추정한다.
3. 진입점이 되는 파일을 찾는다.
   - 예: `index.html`
   - 예: `app.js`, `main.js`, `script.js`
   - 예: `style.css`
   - 예: `data.json`
   - 예: Python의 경우 `main.py`, `app.py`, `server.py`
4. 여러 파일이 함께 작동한다면 파일 간 관계를 설명한다.
5. 코드가 실제로 어떤 순서로 실행되는지 설명한다.
6. 사용자가 화면에서 어떤 행동을 했을 때 어떤 코드가 반응하는지 설명한다.
7. 초보자가 반드시 알아야 할 개념을 뽑아 설명한다.
8. 가독성, 유지보수성, UI/UX, 접근성, 성능, 보안 관점에서 개선점을 제시한다.
9. 마지막에는 오늘 공부할 수 있는 학습 과제와 복습 질문을 제공한다.

## 2. 답변 언어와 설명 수준

답변은 한국어로 해줘.

나는 초보자이므로 전문용어를 그냥 던지지 말고, 처음 나오는 용어는 반드시 쉽게 풀어줘.

예를 들어 다음처럼 설명해줘.

- DOM: HTML 문서를 JavaScript가 조작할 수 있도록 객체처럼 표현한 구조
- 이벤트 리스너: 사용자의 클릭, 입력, 스크롤 같은 행동을 감지하는 코드
- 함수: 반복해서 실행할 수 있도록 이름을 붙여둔 코드 묶음
- 객체: 관련 있는 값을 이름표와 함께 묶어둔 데이터 구조
- 배열: 여러 값을 순서대로 담아두는 목록
- CSS 선택자: 어떤 HTML 요소에 스타일을 적용할지 고르는 문법
- localStorage: 브라우저 안에 간단한 데이터를 저장하는 공간

설명은 “개념 설명 → 현재 코드에서의 실제 예시 → 내가 이해해야 할 포인트” 순서로 해줘.

## 3. 전체 구조 분석 형식

먼저 다음 형식으로 전체 구조를 요약해줘.

### A. 프로젝트 한 줄 요약

이 프로젝트가 무엇을 하는 코드인지 한 문장으로 설명해줘.

### B. 전체 파일 구조 요약

폴더와 파일을 다음 기준으로 분류해줘.

| 분류 | 해당 파일/폴더 | 역할 |
|---|---|---|
| 화면 구조 | HTML 파일 | 페이지의 뼈대 |
| 동작 로직 | JS 파일 | 버튼 클릭, 계산, 데이터 처리 |
| 디자인 | CSS 파일 | 색상, 배치, 반응형 화면 |
| 데이터 | JSON, TXT 등 | 화면에 넣을 원본 데이터 |
| 공통 모듈 | footer, loader, config 등 | 여러 페이지에서 반복 사용 |
| 자산 | 이미지, PDF, 엑셀, 아이콘 등 | 화면 표시 또는 다운로드용 자료 |

### C. 핵심 파일 우선순위

초보자인 내가 어떤 파일부터 읽으면 좋은지 순서를 정해줘.

예시:

1. `index.html` — 화면의 기본 구조를 이해하기 좋음
2. `style.css` — 전체 디자인 규칙을 이해하기 좋음
3. `script.js` — 버튼 클릭과 계산 로직을 이해하기 좋음
4. `data.json` — 화면에 표시되는 원본 데이터 구조를 이해하기 좋음
5. 공통 JS 파일 — 여러 페이지가 공유하는 기능을 이해하기 좋음

각 파일마다 “왜 먼저 봐야 하는지”도 설명해줘.

## 4. 여러 파일 간 관계 분석

한 번에 여러 파일을 올린 경우, 반드시 파일 간 관계를 분석해줘.

다음 형식으로 설명해줘.

### A. 파일 관계도

가능하면 텍스트 다이어그램으로 표현해줘.

예시:

```text
index.html
 ├─ style.css
 │   └─ 화면 배치, 색상, 버튼 스타일 담당
 ├─ script.js
 │   ├─ 버튼 클릭 감지
 │   ├─ 입력값 읽기
 │   ├─ 계산 실행
 │   └─ 결과를 HTML에 표시
 └─ data.json
     └─ 선택지/기준값/표시 문구 제공
```

### B. 실행 흐름

사용자가 웹페이지를 열었을 때부터 기능을 사용하는 순간까지 실행 흐름을 설명해줘.

예시:

1. 브라우저가 `index.html`을 읽는다.
2. HTML 안에서 CSS 파일을 불러와 화면 모양을 만든다.
3. HTML 안에서 JS 파일을 불러온다.
4. JS가 버튼, 입력창, 결과 영역을 찾는다.
5. 사용자가 버튼을 누르면 이벤트 리스너가 실행된다.
6. JS가 입력값을 계산하거나 가공한다.
7. 결과를 다시 HTML 화면에 표시한다.

### C. 데이터 흐름

데이터가 어디서 시작해서 어디로 이동하는지 설명해줘.

예시:

```text
사용자 입력
 → JavaScript에서 값 읽기
 → 함수에서 계산
 → 결과 문자열 생성
 → HTML 결과 영역에 표시
 → 필요하면 localStorage에 저장
```

### D. 의존 관계

어떤 파일이 다른 파일에 의존하는지 설명해줘.

예시:

| 파일 | 의존하는 대상 | 이유 |
|---|---|---|
| `index.html` | `style.css` | 화면 디자인 적용 |
| `index.html` | `script.js` | 클릭, 입력, 계산 기능 실행 |
| `script.js` | `data.json` | 기준 데이터 불러오기 |
| `script.js` | localStorage | 사용자 입력값 저장 |

## 5. HTML 분석 기준

HTML 파일을 분석할 때는 다음을 봐줘.

1. 페이지의 큰 구조
   - header
   - nav
   - main
   - section
   - article
   - footer
2. 제목 구조
   - h1, h2, h3가 논리적으로 쓰였는지
3. 입력 요소
   - input
   - select
   - textarea
   - button
   - form
4. 결과 표시 영역
   - 계산 결과
   - 안내문
   - 표
   - 카드
5. 접근성
   - label이 있는지
   - 버튼 이름이 명확한지
   - 스크린리더가 읽기 좋은지
   - 키보드만으로도 사용할 수 있는지
6. 유지보수성
   - id와 class 이름이 이해하기 쉬운지
   - HTML이 너무 길거나 반복이 많은지
   - JS와 CSS가 지나치게 섞여 있지 않은지

초보자가 이해할 수 있도록 “이 태그는 왜 쓰였는지”, “다른 태그를 쓰면 어떤 차이가 있는지”도 설명해줘.

## 6. JavaScript 분석 기준

JavaScript 파일을 분석할 때는 다음을 봐줘.

1. 전체 코드의 역할
2. 주요 변수
3. 주요 함수
4. 이벤트 리스너
5. DOM 조작
6. 계산 로직
7. 조건문
8. 반복문
9. 배열과 객체
10. localStorage 사용 여부
11. fetch 또는 외부 데이터 호출 여부
12. 에러 처리
13. 전역 변수 남용 여부
14. 함수가 너무 길거나 역할이 많은지
15. 같은 코드가 반복되는지
16. 사용자가 잘못 입력했을 때 안전하게 처리하는지

각 함수는 다음 형식으로 설명해줘.

### 함수명: `함수이름`

| 항목 | 설명 |
|---|---|
| 역할 | 이 함수가 하는 일 |
| 입력값 | 함수가 받는 값 |
| 처리 과정 | 내부에서 어떤 순서로 처리하는지 |
| 결과 | 화면이나 데이터에 어떤 변화가 생기는지 |
| 초보자 학습 포인트 | 이 함수로 배울 수 있는 개념 |
| 개선 가능성 | 더 읽기 쉽게 만들 수 있는 방법 |

함수 설명 후에는 가능하면 의사코드도 제공해줘.

예시:

```text
이 함수의 흐름을 사람 말로 바꾸면 다음과 같다.

1. 입력창에서 값을 가져온다.
2. 값이 비어 있으면 경고를 보여준다.
3. 값이 있으면 숫자로 바꾼다.
4. 계산식을 적용한다.
5. 결과 영역에 문장을 만들어 넣는다.
```

## 8. Python 분석 기준

Python 코드가 포함된 경우 다음을 봐줘.

1. 이 Python 파일이 웹페이지와 직접 관련 있는지, 보조 도구인지 구분해줘.
2. 실행 진입점이 어디인지 찾아줘.
3. 주요 함수와 클래스의 역할을 설명해줘.
4. 파일 읽기/쓰기, 엑셀 처리, PDF 처리, 데이터 변환 등이 있으면 흐름을 설명해줘.
5. 외부 라이브러리를 사용한다면 라이브러리 역할을 설명해줘.
6. 에러 처리와 예외 처리가 있는지 봐줘.
7. 초보자가 배울 만한 Python 문법을 뽑아줘.
8. Python 파일을 실행하려면 어떤 순서로 해야 하는지 알려줘.

가능하면 다음 형식으로 정리해줘.

```text
실행 흐름:
1. 필요한 라이브러리를 불러온다.

3. 데이터를 가공한다.
```

## 9. UI/UX와 사용환경 분석

웹페이지나 화면이 있는 경우, 반드시 UI/UX 관점도 분석해줘.

내가 만드는 웹페이지는 실제 사용자가 빠르게 업무를 처리하기 위해 쓰는 경우가 많다. 특히 한국어 업무 문서, 행정 업무, 교직원, 실무자, PC 중심 사용, 가끔 모바일 사용, 출력/PDF 저장 같은 상황을 고려해줘.

다음 항목을 확인해줘.

1. 첫 화면에서 사용자가 무엇을 해야 하는지 바로 알 수 있는가
2. 버튼 이름이 명확한가
3. 입력창의 의미가 분명한가
4. 안내문이 너무 길거나 추상적이지 않은가
5. 표가 모바일에서 깨지지 않는가
6. 글자 크기가 충분한가
7. 색상 대비가 충분한가
8. 중요한 정보가 눈에 잘 띄는가
9. 사용자가 실수했을 때 복구할 수 있는가
10. 결과를 복사, 저장, 인쇄, PDF로 내보내기 쉬운가
11. 고령 사용자나 비개발자도 이해할 수 있는가
12. 행정 문서처럼 정확성이 중요한 화면에서 오해를 줄이고 있는가

UI/UX 개선점은 다음 형식으로 제시해줘.

| 항목 | 현재 상태 | 문제 가능성 | 개선 제안 | 우선순위 |
|---|---|---|---|---|

우선순위는 `상`, `중`, `하`로 표시해줘.

## 10. 가독성과 유지보수성 분석

코드를 읽기 쉽고 나중에 고치기 쉬운지 봐줘.

다음 기준으로 분석해줘.

1. 파일이 너무 큰지
2. 함수 하나가 너무 많은 일을 하는지
3. 변수명과 함수명이 의미 있는지
4. HTML, CSS, JS의 역할이 잘 분리되어 있는지
5. 같은 코드가 반복되는지
6. 설정값이 코드 곳곳에 흩어져 있는지
7. 주석이 필요한 곳에 있는지
8. 주석이 너무 많거나 오래되어 오히려 방해되는지
9. 삭제해도 될 것 같은 코드가 있는지
10. 확장하기 쉬운 구조인지
11. 여러 페이지에서 반복되는 공통 요소를 잘 관리하고 있는지
12. 데이터와 화면 표현이 적절히 분리되어 있는지

개선 제안은 반드시 초보자가 이해할 수 있게 단계적으로 해줘.

나쁜 예:
“컴포넌트화하세요.”

좋은 예:
“현재 같은 버튼 HTML이 여러 페이지에 반복되어 있습니다. 나중에 버튼 문구나 스타일을 바꾸려면 여러 파일을 동시에 고쳐야 합니다. 공통 버튼 생성 함수를 만들거나, 공통 CSS class를 만들어두면 유지보수가 쉬워집니다.”

## 11. 보안, 안정성, 개인정보 관점

웹페이지나 업무도구라면 다음도 확인해줘.

1. 사용자가 입력한 값을 그대로 HTML에 넣고 있지는 않은가
2. `innerHTML` 사용이 위험하지 않은가
3. localStorage에 민감한 정보가 저장되지는 않는가
4. 외부 CDN을 사용할 때 장애나 보안 리스크가 있는가
5. 파일 업로드, 엑셀 처리, PDF 처리 기능이 있다면 예외 처리가 충분한가
6. 사용자가 이상한 값을 입력했을 때 화면이 깨지지 않는가
7. 개인정보나 업무상 민감 정보가 코드에 하드코딩되어 있지 않은가
8. API 키, 토큰, 비밀번호 같은 비밀값이 포함되어 있지 않은가

단, 나는 초보자이므로 겁주듯이 말하지 말고, “왜 문제가 될 수 있는지”와 “현실적으로 지금 할 수 있는 개선 방법”을 같이 알려줘.

## 12. 성능 관점

다음 성능 요소도 봐줘.

1. 이미지 파일이 너무 크지 않은가
2. JS 파일이 불필요하게 많이 로드되지 않는가
3. 모든 페이지에서 필요 없는 라이브러리를 불러오고 있지는 않은가
4. 외부 CDN 의존성이 많은가
5. 모바일에서 느릴 가능성이 있는가
6. PDF, 엑셀, 이미지 처리 기능이 브라우저에 과도한 부담을 주지 않는가
7. 반복문이나 이벤트가 불필요하게 자주 실행되지는 않는가
8. 초기 화면 표시 속도를 늦추는 요소가 있는가

개선점은 “당장 해야 할 것”과 “나중에 해도 되는 것”으로 나눠줘.

복사용 프롬프트

내 현재 코드와 개발 지식 수준을 기준으로 추천 학습 순서를 다시 잡아줘.

현재 내 상황은 다음과 같아.

- 개발 지식은 아직 거의 없음
- HTML, CSS, JavaScript, Python을 실제 운영 중인 코드로 배우고 싶음
- AI 도움으로 웹페이지와 업무 도구를 만들어왔음
- 전체 구조를 보는 감각은 있으나, 세부 문법과 실행 흐름은 아직 약함
- 매일 아침 30~60분씩 공부할 예정

다음 순서를 기준으로, 내 코드에서 어떤 파일을 보면 좋을지 추천해줘.

1. `index.html` — 웹페이지의 뼈대 이해
2. 공통 `style.css` — 전체 화면 디자인 규칙 이해
3. 작은 기능의 `script.js` — JS 이벤트와 DOM 조작 이해
4. `data.json` 또는 `*-data.js` — 데이터와 화면 분리 구조 이해
5. `localStorage`를 쓰는 기능 — 브라우저 저장 구조 이해
6. 공통 loader/footer/config 파일 — 여러 페이지에 반복 적용되는 구조 이해
7. PDF/Excel 관련 도구 — 외부 라이브러리와 복잡한 브라우저 기능 이해
8. 큰 계산기/업무도구 — 조건문, 함수 분리, 예외 처리 종합 학습

각 단계마다 다음을 알려줘.

- 왜 이 순서가 좋은지
- 내 코드에서 볼 만한 파일 후보
- 그 파일로 배울 수 있는 개념
- 30~60분 공부 방법
- 복습 질문 3개
